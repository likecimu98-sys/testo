# Стили и сборка CSS — заметка для разработки

Фаза 0 рефакторинга стилей. Цель — снять «заморозку» Tailwind и завести единый
«пульт» для цветов, не меняя текущий внешний вид приложения.

## Что сделано (Фаза 0)

1. **Восстановлена сборка Tailwind.** Раньше `output.css` был «заморожен» (не было
   конфига/`package.json`), и новые классы вроде `z-[10050]` молча не работали.
   Теперь:
   - `npm install` — ставит Tailwind v3.
   - `npm run build:css` — пересобирает `output.css` из `input.css`.
   - `npm run watch:css` — пересборка на лету при правках.
   - `tailwind.config.js` — `darkMode: 'class'`, сканирует `index.html`, `cram.html`, `*.js`.

2. **Добавлены дизайн-токены** — `tokens.css`: палитра CSS-переменных (`--c-brand`,
   `--c-card`, `--c-danger`, … + тёмная тема через `.dark`). Подключён первым в `index.html`.
   Пока **чисто аддитивно** — переменные объявлены, но ещё ничего ими не перекрашено,
   поэтому внешний вид не изменился.

## Важно про `output.css`

Текущий `output.css` — проверенный рабочий артефакт. Контрольная пересборка показала,
что свежий билд **эквивалентен** ему по вычисленным стилям реальных элементов и при этом
на ~900 строк меньше (вычищаются «мёртвые» классы от удалённого кода). Тем не менее
перед тем, как заменить `output.css` пересобранным, прогоните визуальную проверку
**7 скинов × светлая/тёмная** на ключевых экранах (лобби, классика, ДЗ, композёр,
аналитика). Это первый шаг Фазы 1.

`cram.html` (Зубрёжка) — самостоятельный, со своими изолированными стилями, в эту
систему не входит и не требует пересборки.

## Что сделано (Фаза 1)

1. **Починен «мёртвый» цвет `examBlue`.** Кастомный Tailwind-цвет `examBlue`
   (активная вкладка, акцентные числа, год во флешкартах) не попал в `output.css`,
   поэтому в СВЕТЛОЙ теме классы `text-/border-/bg-examBlue` не красили ничего.
   Добавлены утилиты в `styles.css`, привязанные к токену `--c-brand-strong`.
   Тёмная тема не затронута (там приоритет у явных `dark:text-blue-400/300`).

2. **Устранён дубль стилей слотов/фишек.** Раньше `.dnd-slot`/`.dnd-chip`/детектив/
   бумага дублировались в inline-`<style>` внутри `index.html` И в `styles.css`,
   причём inline грузился последним и «выигрывал» — правки в `styles.css` молча
   не действовали. Теперь весь этот пласт удалён из inline (там остались только
   no-select и экран-заставка), а единственный источник — `styles.css`.
   «Сильные» правила верных/неверных слотов (высокая специфичность против скинов)
   перенесены в `styles.css` без изменения внешнего вида.

3. **Акцентная палитра в `styles.css` токенизирована.** Все вхождения бренда/
   семантики (`#3b82f6`, `#2563eb`, `#10b981`, `#ef4444`, `#f59e0b`, `#34d399`,
   `#f43f5e`) заменены на `var(--c-*)`. Эти токены тема-инвариантны (блок `.dark`
   в `tokens.css` их не переопределяет), поэтому замена — ноль визуальных изменений,
   но теперь акцент игрового поля управляется из `tokens.css` (живой «пульт»).

## Что сделано (Фаза 2 — акценты в JS)

Акцентные hex в JS-генерируемом интерфейсе токенизированы: в `ui.js`,
`firebase-sync.js`, `visual-trainer.js`, `state.js` все вхождения бренда/семантики
(`#3b82f6`, `#2563eb`, `#10b981`, `#ef4444`, `#f59e0b`, `#8b5cf6`, `#f43f5e`,
`#fde047`) внутри `style="…"` заменены на `var(--c-*)`. Токены тема-инвариантны →
ноль визуальных изменений (проверено: inline-`var()` резолвится в те же rgb).

Тонкость с `firebase-sync.js`: там есть экспорт в PDF (jsPDF), но цвет туда
передаётся ТОЛЬКО числовыми RGB (`setFillColor(37,99,235)`, `setTextColor(...rgb)`),
а hex-строки живут лишь в `style=` и в сравнениях `accColor === '#hex'` → RGB-массив.
Глобальная замена меняет и литерал, и сравнение синхронно, поэтому логика PDF и
его цвета не меняются. `modes.js` намеренно пропущен (своя палитра задач с
кастомными оттенками `#059669/#9333ea/#d97706`).

## Аудит Фазы 1b (скины) — РЕФАКТОРИНГ НЕ НУЖЕН

Проверил `theme-aurora.css` и выяснил: это **уже грамотная токенная система**, а не «грязный угол».

- `:root { --a-bg/--a-s1/--a-b1/--a-txt/… }` — палитра темы по умолчанию (aurora).
- Общий слой перекрытий мапит Tailwind-классы на токены: `.bg-white { background: var(--a-s1) !important }`, `.text-gray-800 { color: var(--a-txt) !important }` и т.п.
- Каждый из 8 скинов — это `body.skin-X { --a-*: … }` (свет) + `html.dark body.skin-X { --a-*: … }` (тьма), т.е. **просто переопределение палитры**; слой перекрытий подхватывает значения автоматически. Плюс по 6-10 точечных правил на скин (шапка/навбар/слот/фишка) с руками подобранными значениями.

Про 653 `!important`: они **структурные**, не мусор — скины перебивают готовые Tailwind-утилиты из `output.css`, а без `!important` утилиту не перебить. Убрать их = переписать всю разметку на `var(--…)` вместо классов цвета (тысячи мест) — огромный риск, нулевая польза. Скины уже «задают только токены» (`--a-*`) — цель 1b фактически достигнута by design.

**Вывод: `theme-aurora.css` не трогаем.** Рефакторинг стилей считаем завершённым.
